01 - Introduction

analyse

出现时间 angular->react->vue
angular内置了很多东西,很全面
react只提供ui相关,功能少,其他功能都是自由组合(第三方)
vue希望结合其他两个框架的优点,提供了状态管理和路由等功能

angular适合大型公司,因为它配套齐全
react适合初创公司,它可以自由选择生态工具链(当然,选择太多也会导致项目混乱)
vue结合了两者的优点

composition vs options api

016 Computed Properties–[koudaizy.com]

如果在method里计算fullname,则当其他data的数据更新时,fullname也会被重新计算,因为每次更新数据,vue都会重新渲染数据到页面上,则会调用所有method方法(由事件触发的方法不会),所以,我们需要计算属性来让方法在需要时才被调用

const vm = Vue.createApp({
    // 定义数据
    data() {
        return {
            firstName: 'John',
            middleName: '',
            lastName: 'Bob', 
        }
    },
    methods: { 
        fullName() {
            console.log(1);
            return `${this.firstName} ${this.middleName} ${this.lastName.toUpperCase()}`
        }, 
    },
})
    // 挂载vue到id=app的元素
    .mount('#app')

添加计算属性

只有和计算属性关联的值的更新,才会触发计算属性的更新

computed: {
    fullName() {
        console.log(1);
        return `${this.firstName} ${this.middleName} ${this.lastName.toUpperCase()}`
    },
}

005 Understanding Reactivity with Proxies–[koudaizy.com]

excel中响应式的例子: 由两个值计算某个值并显示在页面, 当前两个值更新, 它们计算得出的值也动态更新

vue3 响应式使用的是proxy代理对象(vue2 -> defineProperty)

const data = {
    name: ''
}
// proxy 返回的是对象的副本
const observedData = new Proxy(data, {
    // 值更新时触发
    set(target, key, value) {
        document.querySelector('#name').innerHTML = value
        target[key] = value
    }
})
observedData.name = 'lihua'
console.log(observedData.name);
